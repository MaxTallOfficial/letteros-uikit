# Правила композиции

Как использовать токены и компоненты UI kit без ухода в редизайн.

## Основной принцип

Собирай страницы из того, что есть. Не придумывай новые значения, если существующий токен подходит. Не обобщай различия, если они зафиксированы.

## Безопасные комбинации

### Кнопки

- Любой `variant` × любой `size` — валидная комбинация
- На тёмном фоне: `white` или `transparentWhite`
- На светлом фоне: `blue`, `black`, `transparentBlack`, `transparentBlue`, `green`
- На pricing-страницах: `blueNew`, `transparentBlueNew`
- `full` можно добавить к любой комбинации

### Типографика

- Выбирай уровень по контексту страницы, а не по семантике тега
- Для страницы в стиле главной: `h1Hero`, `h2Sections`, `body`
- Для pricing-контекста: `h1Pricing`, `h2PricingBanner`, `bodyPricing`
- Не микшируй pricing-уровни на обычных страницах и наоборот

### Карточки

- Каждый тип карточки предназначен для своего контекста
- `differents` — фичи, преимущества
- `pricing` — тарифные планы
- `blog` — статьи, посты
- `hows` — шаги, процессы (на градиентном фоне)
- Не используй `blog`-карточку для фич и наоборот

### Формы

- Input + Button (`black`, `m`, `full`) = паттерн подписки
- Input + Select + Textarea + FileUpload = паттерн формы заказа
- Checkbox `tiny` — для legal-текстов под формами
- Checkbox `large` — только если есть аналог на /templates/

## Где нельзя делать ложные обобщения

### Два синих — не один

Никогда не используй `#3072ED` вне pricing-контекста. Никогда не используй `#001DFF` для pricing-new элементов. Это параллельные системы.

### Типографика — не шкала

H2 имеет 4 размера не потому что это 4 уровня важности, а потому что на разных страницах H2 стилизован по-разному. При создании новой страницы — выбери ближайший контекст, не изобретай промежуточный размер.

### Карточки — не одна с вариантами

6 типов карточек — это 6 разных паттернов с разными radius, shadow и border. Не пытайся сделать `<Card type="custom" radius={25}>`.

### Link hover — контекстно-зависим

Hover-механизм зависит от того, где находится ссылка (header, footer, content). Не применяй footer-hover в header и наоборот.

## Page-specific distinctions

Некоторые компоненты и токены привязаны к конкретным страницам:

| Сущность | Привязка |
|---|---|
| `blueNew`, `blueNewHover` | Только /pricing-new/ |
| `h2PricingFaq`, `h2PricingBanner` | Только pricing |
| `checkboxLarge` | Только /templates/ |
| PricingNav | Только pricing table |
| Toggle (месяц/год) | Pricing, но потенциально reusable |

Если создаёшь новую страницу, не тяни pricing-specific токены на неё без явного решения.

## Как не уйти в редизайн

1. Не добавляй новые цвета — используй существующую палитру
2. Не изобретай промежуточные размеры типографики
3. Не меняй radius карточек — каждый тип имеет свой фиксированный
4. Не добавляй тени к элементам, которые на сайте flat
5. Не меняй transition-значения — 0.25s для кнопок, 0.2s для ссылок
6. Если нужен новый паттерн — зафиксируй пробел (см. workflow)
