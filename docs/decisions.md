# Решения по UI kit

## 1. AS-IS, не нормализация

UI kit собран строго по реальным стилям сайта letteros.com.
Никакие значения не улучшены и не нормализованы.
Если на сайте два разных синих — оба в токенах.

## 2. Typography — не абстрагирована

На сайте H2 используется с 4 разными размерами (88px, 64px, 56px, 48px).
Вместо одного `h2` в токенах есть `h2Sections`, `h2PricingBanner`, `h2PricingSection`, `h2PricingFaq`.
Так же body / bodyPricing / bodyFaqAnswer.

## 3. Cards — не склеены

Каждый тип карточки (differents, hows, integrations, pricing, blog, faq) сохранён отдельно.
Они реально различаются по radius, shadow, padding, border.

## 4. Два слоя токенов

- `src/tokens/index.ts` — TypeScript-константы для компонентов и как source of truth
- `src/app/globals.css` — CSS-переменные для использования в Tailwind и чистом CSS

Оба слоя зеркалят одни и те же значения из аудита.

## 5. UI kit page — собрана из компонентов

Страница `/uikit` не захардкожена: она импортирует компоненты и рендерит их
с реальными props. Если компонент изменится, витрина обновится автоматически.

## 6. Что осознанно не включено

- Header/Footer/Nav — слишком привязаны к навигационной структуре CMS
- Modal — DOM-структура не была извлечена при аудите
- Brands Marquee — требует реальных лого
- Swiper/Slider — сторонняя библиотека, не часть дизайн-системы

Стили этих элементов задокументированы в audit.md и могут быть реализованы
из существующих примитивов при необходимости.
