# Зафиксированные решения

## 1. AS-IS, не нормализация

UI kit собран строго по реальным стилям letteros.com. Никакие значения не улучшены и не нормализованы. Это осознанный выбор: UI kit должен отражать реальность, а не идеал.

## 2. Два синих акцента сохранены параллельно

`#001DFF` (основная система) и `#3072ED` (только pricing-new) — разные цвета с разным поведением hover. Они не были объединены, потому что на сайте это две независимые системы с отдельными CSS-классами.

## 3. Типографика не абстрагирована

H2 существует в 4 размерах (88/64/56/48px), body — в 3 вариантах line-height. Вместо обобщения каждый вариант сохранён как отдельный уровень (`h2Sections`, `h2PricingBanner` и т.д.). Причина: непонятно, какие различия осознанные, а какие случайные. Безопаснее сохранить все.

## 4. Карточки — 6 отдельных типов

differents, hows, integrations, pricing, blog, faq — каждый тип имеет уникальную комбинацию radius/shadow/border/padding. Попытка создать универсальную карточку потеряла бы эти различия.

## 5. Toggle и Pricing Nav не объединены

Toggle — переключатель двух состояний (radius 32px, generic). Pricing Nav — навигация по тарифам внутри таблицы сравнения (border-bottom, привязан к pricing). Разные паттерны, разные компоненты.

## 6. Checkbox — 4 варианта без слияния

default (14px/#A9A9A9), simple (14px/#151515), tiny (10px), large (24px/500). Large существует только на /templates/ и принципиально отличается от остальных.

## 7. Link hover — три механизма зафиксированы отдельно

- Header/dropdown: `::before` pseudo-underline
- Footer: `text-decoration: underline` + color
- Content/cards: только color change

В React-компонентах каждый контекст реализует свой hover, без универсального хелпера.

## 8. Header, Footer, MobileMenu — реальная структура

Эти компоненты содержат реальный контент и структуру Letteros (навигация, ссылки, лого). Они не абстрагированы в конфигурируемые шаблоны, а воспроизводят сайт as-is.

## 9. SVG-иконки извлечены из production

LogoMark, LogoWordmark, TelegramIcon, VkIcon — реальные path data из letteros.com. Серые варианты логотипа используют `rgb(169, 169, 169)` для footer.

## 10. Section padding не подчиняется шкале

Значения от 40px до 120px без явной spacing scale. Сохранены as-is как `sectionPaddingBottom`. Это не ошибка — так устроен реальный сайт.

## 11. Input font-size варьируется

18px на /templates/, 16px в footer и других контекстах. Оба варианта зафиксированы.

## 12. Что осознанно не включено

- **Pricing Comparison Table** — слишком специфичный layout, зависит от данных
- **Hero Section** — контекстно уникален на каждой странице
- **Swiper/Slider** — сторонняя библиотека, не часть дизайн-системы
- **btn-loader** — изолированное решение, не часть основной кнопочной системы
