# Workflow: создание новых страниц

Пошаговая инструкция для агентов и разработчиков.

## Шаг 1. Изучить систему

Перед началом работы обязательно прочитать:
- `CLAUDE.md` — правила и ограничения
- `docs/design-tokens.md` — доступные токены
- `docs/components-inventory.md` — доступные компоненты
- `docs/composition-rules.md` — что с чем сочетается

## Шаг 2. Определить контекст страницы

Новая страница ближе к:
- **Главной** → использовать `h1Hero`, `h2Sections`, `body`, `accent.blue`
- **Pricing** → использовать `h1Pricing`, `bodyPricing`, допустимо `accent.blueNew`
- **Blog/контентная** → использовать `body`, `small`, карточки `blog`
- **Форма/заказ** → использовать FormInput, Select, Textarea, Checkbox

Контекст определяет, какие токены типографики и акцента использовать.

## Шаг 3. Собрать страницу из существующих элементов

1. Обернуть в `<Container>` для правильных max-width и padding
2. Использовать `<Typography>` с подходящим уровнем
3. Подобрать карточки по типу контента
4. Использовать кнопки с правильным variant и size
5. Соблюдать spacing из токенов (gap, padding)

## Шаг 4. Проверить на соответствие системе

- Все цвета — из `colors`?
- Все размеры шрифтов — из `typography`?
- Все radius — из `radius`?
- Нет самодельных теней, отступов, цветов?
- Правильный синий для контекста (`blue` vs `blueNew`)?

## Шаг 5. Если нужного паттерна нет

**Нельзя** молча придумать новый компонент или токен.

Порядок действий:
1. Проверить, нет ли подходящего существующего паттерна
2. Если нет — зафиксировать пробел в `docs/open-questions.md`
3. Предложить осторожное расширение, обосновав его
4. Дождаться подтверждения перед реализацией
5. Если расширение принято — добавить токен/компонент и обновить docs

## Шаг 6. После завершения

1. Проверить билд (`npm run build`)
2. Визуально проверить результат
3. Если были добавлены новые токены или компоненты — обновить:
   - `docs/components-inventory.md`
   - `docs/design-tokens.md`
   - `docs/decisions.md` (если принято архитектурное решение)

## Чеклист перед сдачей

- [ ] Страница использует только существующие токены и компоненты
- [ ] Контекст страницы определён (главная / pricing / blog / форма)
- [ ] Правильный синий акцент для контекста
- [ ] Нет самовольных новых значений
- [ ] Билд проходит
- [ ] Документация обновлена (если были изменения в системе)
