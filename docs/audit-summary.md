# Выжимка аудита

Полный аудит: `audit.md` в корне репозитория.

## Общая визуальная система

Минималистичный flat-дизайн: почти чёрный текст (`#151515`) на белом фоне, один ярко-синий акцент (`#001DFF`), шрифт Raleway. Тени почти не используются (только на feature/faq-карточках). BEM-архитектура, единый CSS-файл без custom properties.

## Ключевые особенности

- **Один шрифт** — Raleway, веса 400/500/600/700
- **Flat-дизайн** — `box-shadow: none` на большинстве элементов
- **BEM-именование** в оригинале, в UI kit — React-компоненты с inline styles
- **Кнопочная система** — две оси: размер (`ss`/`s`/`m`) × цвет (9 вариантов)
- **Transition-паттерн** — 0.25s для кнопок, 0.2s для ссылок
- **Container** — max-width 1280px, responsive сужение

## Критичные distinctions (нельзя искажать)

### Два параллельных синих

| Цвет | Где | Hover |
|---|---|---|
| `#001DFF` | Весь сайт | Светлеет → `#4B60FF` |
| `#3072ED` | Только `/pricing-new/` | Темнеет → `#0A51D7` |

Это **две разные системы**, не алиасы. Классы с суффиксом `-new` — отдельная ветка.

### Типографика не унифицирована

- H1: 76px (главная) vs 64px (pricing)
- H2: 88px, 64px, 56px, 48px — четыре разных размера
- Body line-height: 25.2px / 27px / 23.4px — три варианта
- Все сохранены как отдельные уровни, не обобщены

### Карточки — 6 разных типов

differents (20px, тень), hows (40px, градиент), integrations (20px, тень), pricing (20px, border), blog (15px, border), faq (20px, hover-тень). Нельзя объединять.

### Link hover — три разных механизма

- Header/dropdown: `::before` pseudo-underline (анимация высоты)
- Footer: `text-decoration: underline` + смена цвета
- Content/cards: только смена цвета

### Checkbox — 4 варианта

default (14px, серый), simple (14px, чёрный), tiny (10px), large (24px/500). Large — только на `/templates/`.

### Input — варианты по контекстам

- Основной: height 54px, font-size 18px
- Footer: компактнее, font-size 16px

## Что особенно важно не исказить

1. Параллельность двух синих систем и противоположные направления hover
2. Размерное разнообразие типографики — это реальность сайта, не ошибка
3. Различия между карточками — каждый тип имеет уникальные radius/shadow/border
4. Различия в механизмах hover у ссылок в разных контекстах
5. Toggle и Pricing Nav — разные паттерны, не один компонент
