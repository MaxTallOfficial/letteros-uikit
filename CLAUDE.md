# Letteros UI Kit — инструкции для агента

## Что это

Автономный UI kit проект для сайта letteros.com. Содержит дизайн-токены и React-компоненты, извлечённые из production-сайта через аудит. Является **source of truth** для создания новых страниц Letteros.

## Жёсткие правила

### Letteros as-is

- UI kit отражает интерфейс Letteros **как он есть**. Не улучшай дизайн по своей инициативе.
- Не нормализуй различия между страницами — они зафиксированы сознательно.
- Не меняй значения токенов, если не получил явное указание.

### Запрет на выдумывание

- **Не придумывай** новые компоненты, токены, цвета или паттерны без явной фиксации.
- **Не теряй distinctions** между реально разными паттернами (два синих, 6 типов карточек, 4 варианта checkbox, 3 механизма link hover).
- **Не объединяй** то, что на сайте существует отдельно.

### Если нужного паттерна нет

1. Проверь, что существующего токена/компонента действительно не хватает.
2. Зафиксируй пробел в `docs/open-questions.md`.
3. Предложи **осторожное** расширение с обоснованием.
4. Дождись подтверждения перед реализацией.

### После значимых изменений

Кратко обнови соответствующие файлы в `docs/` — inventory, tokens, decisions.

## Обязательные файлы для чтения

Перед любой работой по новым страницам или изменениям в системе **изучи**:

| Файл | Что даёт |
|---|---|
| `docs/design-tokens.md` | Все доступные токены |
| `docs/components-inventory.md` | Все доступные компоненты |
| `docs/composition-rules.md` | Что с чем можно сочетать, где нельзя обобщать |
| `docs/page-creation-workflow.md` | Пошаговый процесс создания страниц |
| `docs/decisions.md` | Почему система устроена именно так |
| `docs/open-questions.md` | Что пока не зафиксировано |

Полный аудит: `audit.md`. Контекст проекта: `docs/project-context.md`.

## Source of truth

- **Токены**: `src/tokens/index.ts` + `src/app/globals.css`
- **Компоненты**: `src/components/ui/*.tsx`
- **Документация**: `docs/*.md`
- **Аудит**: `audit.md`

Если документация противоречит коду — код является приоритетом. Обнови документацию.
